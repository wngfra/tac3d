Working with Probes¶

Probes enable you to see how the network changes over time. To create a probe, you first create a monitor (see Working with Monitor) and then establish either a u probe, a v probe or a spike probe on the compartment(s) you wish to monitor. Here's a simple code snippet (which you will find in most tutorials) to get you started:

uProbe = mon.probe(n2Core.cxState, [0], 'u')[0]

fig = plt.figure(1001)

plt.subplot(1, 2, 1)
uProbe.plot()
plt.title('u')

class nxsdk.graph.monitor.probes.PerformanceProbeCondition(tStart, tEnd, bufferSize=1, binSize=1)¶

Represents a ProbeCondition that leads to performance state execution time or energy consumed per algorithmic timestep being recorded at fixed intervals.

Parameters

  • tStart (int) -- Time tStart at which the probing starts

  • tEnd (int) -- Time tEnd at which the probing ends.

  • bufferSize (int) -- Specifies the size of the buffer on the LMT that will be used to collect time stamps. Size in terms of timesteps, e.g. bufferSize = 100, will collect beginning and ending time stamps for 100 timesteps before sending data back to the host.

  • binSize (int) -- Specifies the binSize of the time stamps collected on the LMT, e.g. a binSize of 8 indicates that the time stamps for 8 time steps will be summed in a single bin and will be returned as a single entry.

class nxsdk.graph.monitor.probes.Probe(id, probeCondition)¶

Probes define which states to monitor in a NodeSet. After probes have been generated by the user they are eventually post processed by the compiler. During compilation, a new set of probes is generated based on the addresses of the original probes (e.g. high level probes may map to multiple low-level probes). In addition a map from original to new probes is created. The new probes are then used by the N2Driver during simulation to produce TimeSeries data. After simulation, the TimeSeries data of the new probes is converted into TimeSeries data of the original probes.

Subclasses of Probe must store the address of the probe and implement the getAddress(..) method.

class nxsdk.graph.monitor.probes.SpikeProbeCondition(dt=1, tStart=1)¶

Represents a ProbeCondition that specifies when to start recording spikes (i.e. the time step to start monitoring), and the time interval of which the spikes are accumulated. For example, a probe condition with tStart=1 and dt=1 starts recording spikes at time step 1 and will count the number of spikes at each time step. Where as, a probe condition with tStart=1 and dt=10, will start recording the spikes at time step 1, but will accumulate the spike count over 10 time steps before resetting the count. A value of dt=0 indicates to accumulate the spikes over the course of the entire run.

Note: In case of Payload Probes (GradeSpike, ProbeAxon), we enforce dt == 0 or dt == 1 and tStart = 1
