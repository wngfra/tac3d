Synapses¶

class nxsdk.arch.n2a.graph.nodesets.synapses_obj_inf.SynapsesObjInf(Synapses=0, numStdp=0)¶

Represents Synapses NodeSet which corresponds to an uncompressed list of all synapses on a neuro core. This NodeSet does not correspond to a physical register in a neuro core but only exists as a more convenient user interface to read and write synapses. The synaptic state is actually densely bit-encoded in the SynapseMem NodeSet whose entries are generated from the Synapses NodSet via the synapse compiler:

Synapses --[SynapseCompiler]--> SynapseMem

Each synapse, is described by the absolute compartment index that it is connected to, a weight, delay and tag, a learning-enabled flag and a pointer to a SynapseFmt entry that provides further shared configuration parameters for a synapse.

Synapse organization:

All synapses belonging to an input axon have to be defined sequentially as a sublist within the Synapses NodeSet because the SynapseMap NodeSet points to the beginning and end of this sublist. To optimize synapse encoding, synapses that point to the same SynapseFmt entry should be allocated sequentially. In addition, those synapses should be sorted in ascending order by their compartment index.

To encode Synapses into SynapseMem, the SynapseCompiler follows the following encoding scheme:

  • All synapses of an axon are organized in a set of lists called synaptic entries. With every list being at most 60 synapses long. A new list will be created if the SynapseFmt pointer changes of the length of the current exceeds 60.

  • Every synaptic entry consists of a prefix and a list of at most 60 synapses:

SynEntry = (Prefix, {Syn})

  • The prefix contains a pointer to a shared SynapseFmt entry and a set of configuration parameters. Different synaptic entries can share the same SynapseFmt entry but only the synapses in a SynEntry share the same prefix.

  • Each Synapse encodes the synaptic state information like weights,

delay, etc. in a format dictated by the SynapseFmt.

class nxsdk.arch.n2a.graph.nodesets.synapses_obj_inf.SynapsesObjInfWrapper(SynapsesObjInf, parent, entryId=None)¶

Represents Synapses NodeSetInterface.

property CIdx¶

(uint10) Index of the compartment the synapse is connected to. If population-mode (connection-sharing) is used, cIdx is the relative compartment index with respect to the destination population it is a part of. Otherwise, cIdx is the absolute compartment index within the neuro core.

property Dly¶

(uint6) Delay of the synapse. It allows to delay the accumulation of a spike in the compartment current by a certain number of time steps if SynapseFmt.ignoreDly=0. Otherwise dly acts as a general purpose synaptic variable like tag to be used for learning. The dynamic range of dly is 0..dd..62. But the precision 'dd' depends on Synapse.Fmt.dlyBits: \(dd=2^{6 - numDelayBits}\).

That means the given delay will be rounded and the actual delay per spike is :

  • \(numLsbBits = 6 - numDelayBits\)

\(actDelay=(delay >> numLsbBits) << numLsbBits\)

This corresponds to only preserving the numDelayBits most significant bits. Note: When learning is enabled, dly will saturate at the dynamic range boundaries. Also note that DendAccumCfg.delayBits must be equal or larger than dlyBits.

property LrnEn¶

(uint1) Activates learning and thus modification of synaptic state via the learning engine according to the learning rule assigned to the synapse via the composite stdpProfile pointer defined in StdpPreProfileCfg, SynapseFmt and StdpPostState. Note: This setting of LrnEn must be in agreement with SynapseFmt.learningCfg and LrnEn of all other synapses using the same SynEntry. I.e. different settings of LrnEn within a SynEntry are only allowed if SynapseFmt.learningCfg=3.

property Tag¶

(int9) Tag of the synapse. The tag is similar to the synaptic weight in that it is subject to the same kinds of dynamic range, sign-sharing and precision constraints, except that the tag plays no role during spike accumulation but only serves as a general purpose scratch variable for learning purposes. For the tag, numTagBits and tagBits replace the role of numWgtBits and wgtBits in SynapseFmt.

property Wgt¶

(int9) Weight of the synapse. The dynamic range of a weight is generally in the range of -256..0..255 but with variable precision and the dynamic range can be constraint to only positive or negative values. Depending on SynapseFmt.fanoutType and SynapseFmt.wgtBits, allowed values are constraint to:

fanoutType

dynamicRange

MIXED

-256..dw..254

EXCITATORY

0..dw..255

INHIBITORY

-256..dw..0

Here the precision dw depends on fanoutType and numWeightBits.

\(dw=2^{8 - numWeightBits - IS\_MIXED}\) where numWeightBits is defined by SynapseFmt.wgtBits. That means the given weight will eventually be rounded and the actually accumulated weight per spike is :

  • \(numLsbBits = 8 - numWeightBits - IS\_MIXED\)

  • \(actWeight=(wgt >> numLsbBits) << numLsbBits\)

where \(IS_MIXED = fanoutType==MIXED\).

This corresponds to only preserving the \((numWeightBits - IS_MIXED)\) most significant bits. When learning is enabled, weight will saturate at the dynamic range boundaries.

configure(**kwargs)¶

Allows to configure multiple register fields.

property synFmtId¶

(uint4) Specifies pointer into SynapseFmt and thus selects which synapse format is applied to a synapse. Note: SynapseFmt[0] is illegal.
