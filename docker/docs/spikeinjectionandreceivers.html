Spike Injection and Spike Receivers¶

NxSDK provides BasicSpikeGen and InteractiveSpikeGen processes using which spikes can be injected. BasicSpikeGen should be used to add spikes before running the network while InteractiveSpikeGen should be used to send spikes while the network is running.

Along with Probes, SpikeReceivers could also be used to receive the spikes from compartments (groups) using a callback mechanism in an interactive way. Compartments and CompartmentGroups can connect to these spike receivers. These receivers poll the spike activity every 10ms and notify the caller using callback. The caller can then take further action to send more spikes or just monitor.

Refer to jupyter notebook illustrating a demo Interactive Spike Sender Receiver

net = nx.NxNet()

cp = nx.CompartmentPrototype(vThMant=10, functionalState=nx.COMPARTMENT_FUNCTIONAL_STATE.IDLE)
cx = net.createCompartment(prototype=cp)

probes = cx.probe([nx.ProbeParameter.SPIKE])

connproto = nx.ConnectionPrototype(weight=255)

sp = net.createSpikeGenProcess(numPorts=1)
sp.connect(cx, prototype=connproto)

# Add spikes to be inserted before invoking run
sp.addSpikes([0], [[1, 2, 3, 4, 5]])

si = net.createInteractiveSpikeGenProcess(numPorts=1)
si.connect(cx, prototype=connproto)

net.runAsync(2000)

# Now send spikes interactively
for i in range(50):
    sleep(0.2)
    si.sendSpikes([1], [1])

net.disconnect()

nx.plotProbes(probes)
plt.show()

Connects SpikeInputPortGroup to Compartment or CompartmentGroup

Parameters

  • dst=None -- The destination Compartment/Neuron or Compartment/NeuronGroup

  • prototype=None -- A single ConnectionPrototype or list of ConnectionPrototype objects. If a single object is provided, it is used for all connections in the group. If a list is provided <prototypeMap> must be provided as well.

  • prototypeMap=None -- A 0/1/2D numpy array where each element must be within 0..len(prototype) which specifies the prototype to use for each connection. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

  • connectionMask -- A 0/1/2D numpy binary numpy array which specifies which connection exists within the (numDstNodes x numDstNodes) space. Elements in prototypeMap, weights, delays and tags where <connectionMask> == 0 will be ignored. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

  • name=None -- (string) A descriptive name for the group.

  • weight -- A 0/1/2D numpy array weight matrix. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

  • delay -- A 0/1/2D numpy array delay matrix. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

  • tag -- A 0/1/2D numpy array tag matrix. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

  • enableLearning -- A 0/1/2D numpy array enableLearning matrix. A non-2D array is automatically broadcast into a (numDstNodes x numDstNodes) array.

Returns

class nxsdk.net.process.spikereceiver.SpikeReceiver(net)¶

Spike Receiver enables monitoring of spikes via call back methods.

Spike Receivers are meant to be used along with async (non-blocking) execution of NxNet.

callback(obj)¶

Register a callable object or callback function. Here's an example of a callable object.

class Callable:
    def __init__(self):
        self.data = []

    def __call__(self, *args, **kwargs):
        # args[0] is a 2D list of probe data for all compartments connecting to this receiver
        self.data.extend(args[0][0])

callableObj = Callable()

sr = nx.SpikeReceiver(net)

# This calls callableObj.__call__ with the spike activity data
sr.callback(callableObj)

Note

User Callback functions should be light-weight and should return relatively real-time to not slow down other parts of the network.

Parameters

Callable (obj) -- A Callable object or callback function

Returns

None

connect(dst)¶

Connect SpikeReceiver to a destination. You may call connect method multiple times to attach this receiver to different compartments.

Parameters

Compartment/CompartmentGroup (dst) -- Compartment or CompartmentGroup which connects to this receiver

Returns

None

notify()¶

Notifies the spike receiver to perform the callback. Notify method will push timeseries data fetched since the last callback (i.e. only incremental data is pushed).

User Callbacks should be light-weight and should return relatively real-time to not slow down other parts of the network.
